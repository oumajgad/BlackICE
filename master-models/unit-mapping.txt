################################################################
# UNIT-MAPPING FILE
#
# LINE FORMAT:
#
# 		UNIT_NAME=SOURCE_UNIT_NAME,COUNTRY,SMALLCOUNTER_OVERRIDE_NAME,COUNTER_OVVERIDE_NAME,MAPCOUNTER_OVERRIDE_NAME
#
# If you have counter overrides, the scripts look for them in the custom counter folders!
# If you leave those last two fields blank, the script will attempt to find a custom counter
# in the custom folders with the same base name as the unit itself, otherwise, it will fall
# back to the main counter folders and look for the counter for the source unit.
#
# In other words, if you create counters with the same name as the unit itself, and put them
# in the custom counter folders, you don't have to add those two fields.  They are really only 
# needed in certain cases, like the air units.
#
# NOTES:
#
# Unique Divisions should be one of the following:
# 
# division_hq (for normal units, or those that were tougher but part of inferior militaries
# division_hq_veteran (for tougher units, or those that were considered elite but part of inferior militaries
# division_hq_elite (best of the best!)
#
# Unique Corps and above should be "hq_brigade", "armor_hq_brigade", etc.
#
#
